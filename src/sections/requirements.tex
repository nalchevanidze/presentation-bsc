\section{Requirements}

\begin{frame}\frametitle{Requirements}  

\begin{itemize}
    \li{Maintainable} a Maintainable software can adapt to continuous changes.
    \begin{itemize}
        \item Low boilerplate
        \item Familiarity
        \item Modularity
    \end{itemize}
    \li{Reliability}: We aim to avoid runtime failures. 
    \begin{itemize}
        \item Type safety: deterministic derivation, resolver validity, and schema validity.    
        \item Compliance with GraphQL specifications
    \end{itemize}
    \li{Efficiency} The library must be lazy and avoid known Haskell features causing poor performance.
\end{itemize}

\end{frame}
